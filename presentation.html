What is Bundler?

Bundler is a new way to systematically and repeatably manage Ruby application dependencies.

Setup your app to use your bundle

        require "rubygems"
        require "bundler"
        Bundler.setup

        require "nokogiri"
        # any other requires you might need

Bundle your Gems

        # Get Bundler and an up to date RubyGems environment
        $ gem install bundler
        $ gem update --system
        
        # Get your gems and track your Gemfile
        $ bundle install
        $ git add Gemfile
        
        # Run a command provided by a gem
        $ bundle exec rackup

When you're ready to deploy

        $ bundle lock
        $ git add Gemfile.lock

Ruby Version Manager

How do you work with the ton of Rubies out there?

Most folks either don't worry about it, or they do their best with virtualization or custom setups.

RVM makes working with all the popular Rubies stupid simple.

Install multiple Rubies

        # MRI
        $ rvm install 1.8.7
        $ rvm install 1.9.1
        $ rvm install 1.9.2-head
        
        # Other Rubies
        $ rvm install ree
        $ rvm install rbx
        $ rvm install macruby
        $ rvm install ironruby
        $ rvm install jruby

Deal with Gems

        $ rvm 1.9.2-head
        $ gem install rails -v 2.3.3

        $ rvm gemset create rails222 rails126
        Gemset 'rails222' created.
        Gemset 'rails126' created.

        $ rvm 1.9.2-head@rails222
        $ gem install rails -v 2.2.2

        $ rvm 1.9.2-head@rails126
        $ gem install rails -v 1.2.6

RVM Actions

        # Run a script with all your Rubies
        $ rvm ruby test.rb
        $ rvm 1.9.1,jruby,ree test.rb
        
        # Run your tests across many Rubies
        $ rvm 1.8.6,1.9.1 rake test
        $ rvm 1.9.1,1.9.2-head specs
        
        # Benchmark your script on many Rubies
        $ rvm 1.8.6,1.8.7,1.9.1,ree benchmark increment.rb
